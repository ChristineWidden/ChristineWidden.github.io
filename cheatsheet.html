Key Information

Starting Selenium

from selenium import webdriver
from selenium.webdriver.chrome.service import Service
from webdriver_manager.chrome import ChromeDriverManager
import time

Finding an element by its id and printing the text

item = browser.find_element("xpath", "//*[@id ='words']")
time.sleep(1)
print(item.text)

Getting a specific element from a list

XPath is 1-indexed, so this will return the third list item element from the unordered list.

item = browser.find_element("xpath", "//ul/li[3]")

Known issues

Sometimes Selenium may crash and say that it was unable to click an element. Assuming the XPath is correct, here are two potential causes of this problem and their solutions:

Page elements could be overlapping due to window size, preventing Selenium from accessing the right element. Make sure to maximize the window with "browser.maximize_window()" or "driver.maximize_window()".

The actual position and rendered position of an element might be different. This issue has so far been seen with radio buttons. Regardless of what element you are actually trying to click on, have selenium click the element your browser directs you to when you use Inspect Element. For instance, you may be trying to click a radio button, but inspect element tells you that you ACTUALLY should have Selenium try to click on the button's label.
